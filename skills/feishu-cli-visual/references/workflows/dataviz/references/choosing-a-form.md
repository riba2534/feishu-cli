# 选择图表形式 —— 在配色之前决定

数据的**任务**决定形式，不是用户嘴里的图表名决定形式。用户说"画个饼图"，
但数据是 12 个类别的相近占比时，正确答案是横向条形图或表格 —— 先按本文判断，
再礼貌地给出更合适的形式。

## 第一问：这是图表吗？

| 数据是…… | 用 | 而不是 |
|---|---|---|
| 一个当前值（可带趋势） | **大数字**（值 + 增减 + 迷你趋势线） | 只有一根柱子的柱状图 |
| 几个头条数字 | **指标行**（多个大数字并排） | 分组柱状图 |
| 单个比率对上限 | **进度条/仪表**（同 ramp 轨道） | 两片的饼图 |
| 超过 ~7 个都有意义的类别 | **表格**（或表格 + 图） | 更多颜色 |

飞书落地：卡片里大数字用 column_set + 加粗大字号文本 / fields；文档里用
Callout + 加粗数值；htmlbox 里用 DOM 大数字（≥48px，与正文同一无衬线字体）。

## 第二问：任务 → 形式

| 任务（读者要做什么） | 默认形式 | 配色职能 |
|---|---|---|
| 比较量级（低→高） | 柱/条形图；网格数据用热力图 | sequential（单色相） |
| 看时间趋势 | 折线图；单系列可用面积图 | sequential 或 1 个 categorical |
| 区分不同系列 | 分组/堆叠柱、多折线 | **categorical** |
| 一个系列是主角，其余是背景 | **强调式**（主角上色，其余灰） | 1 个主色 + 灰 |
| 高于/低于基线、与目标的偏差 | diverging 条形图、带基线折线 | diverging |
| 部分-整体 | **堆叠条形**（类别多/名字长时横向） | categorical |
| 有序档位占比（满意度、评分分布） | **diverging 堆叠条**，中点对齐中性档 | diverging |
| 每项的前→后变化 | 哑铃图 | 单色相双深浅 |
| 流程/架构/关系（非数据量） | 不是数据图 —— 走 board/Mermaid 结构图 | 浅底/深边派生对 |

## 背后的规则

- **sequential 是安全默认**。单色相、越多越深，最难被误读。除非任务明确是
  "身份"或"极性"，否则先想 sequential。
- **categorical 只在"系列本身就是主题"时用** —— 它有真实代价：会淹没真正重要的
  那个数据点。如果故事是"这一个涨了"，那是**强调式**，不是 categorical。
- **强调式是最被低估的形式**。一个系列穿主色，其余穿灰。"把这张图做清楚点"的
  诚实答案往往就是它。
- **饼图只用于"一眼看出部分-整体"**，≤ 6 片；比较相近值用条形图或直接放数字。

## 系列数阶梯（categorical）

| 系列数 | 处理 |
|---|---|
| 1–3 | 颜色即可分辨；直接标签 |
| 4 | 柱/线（相邻取色已由色板顺序保障）继续按序取；散点/气泡等任意相邻场景已达 all-pairs 安全子集上限（见 palette.md），直接标签成为义务 |
| 5–6 | 软上限；上图例或拆小倍数 |
| 7–8 | 硬上限；再多就折叠"其他"、拆小倍数、或复合编码（色相 × 形状） |

**永远不用"再生成一个色"解决系列过多** —— 第 9 个色在色盲模拟下必然与已有
某色撞车，破坏整个色板的可分性。

## 形式定了之后

回到 `SKILL.md` 选管线（动图 htmlbox / 卡片 card / 文档 import / 画板 board），
然后按 `palette.md` 配色并跑校验。
